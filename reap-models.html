What is REAP?

REAP removes redundant MoE experts by scoring on router gate-values and activation norms. One-shot, no fine-tuning. Drop-in with vanilla vLLM.

Key result: 50% pruning retains 97.6% coding ability, 96.7% SWE-bench. Pruning beats merging on all generative tasks.

Max KV Cache & Context on 96 GB

How this works: 96 GB total minus model weights minus 8 GB OS = RAM available for KV cache. Max context = available RAM / per-token KV cost. Per-token KV cost depends on attention layers, KV heads, and head dimension.

Per-token KV cost (FP16): attn_layers × kv_heads × head_dim × 2 bytes

Deployment

All Cerebras REAP models work with vanilla vLLM. No patches.

# Qwen3-Coder-REAP
vllm serve cerebras/Qwen3-Coder-REAP-363B-A35B-FP8 \
  --tensor-parallel-size 8 \
  --tool-call-parser qwen3_coder \
  --enable-auto-tool-choice --enable-expert-parallel

# MiniMax-M2-REAP
vllm serve cerebras/MiniMax-M2-REAP-162B-A10B \
  --tensor-parallel-size 8 \
  --tool-call-parser minimax_m2 \
  --reasoning-parser minimax_m2_append_think \
  --trust-remote-code
